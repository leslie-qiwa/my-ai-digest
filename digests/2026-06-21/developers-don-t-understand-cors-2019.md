# Developers don't understand CORS (2019)

# 开发者不理解CORS

在全栈咨询领域工作最大的好处之一是，我有机会与来自不同规模和行业公司的大量开发者合作，他们的技能水平各不相同。这让我有机会观察到哪些是普遍存在的困惑。其中一个最近看起来普遍且值得关注的问题是：太多Web开发者不理解CORS的工作原理。

指出这一点似乎恰逢其时，因为最近Zoom曝出了一个漏洞。安全研究员Jonathan Leitschuh发现Zoom在本地机器上运行了一个监听 `http://localhost:19421` 的Web服务器。当你打开一个Zoom链接时，Zoom的网站会向这个本地Web服务器发送请求，告诉它打开原生Zoom应用。整篇文章都值得一读，但以下部分引起了我的注意：

> 我还发现，这个页面并没有发送常规的AJAX请求，而是从本地运行的Zoom Web服务器加载一张图片。图片的不同尺寸决定了服务器的错误/状态码。你可以在这里看到相关的case-switch逻辑。

> 我提出的一个问题是，为什么这个Web服务器要把数据编码在图片文件的尺寸中返回？原因是，这样做是为了绕过跨域资源共享（CORS）。出于非常刻意的原因，浏览器对运行在localhost上的服务器明确忽略任何CORS策略。

最后一句话是不正确的——Chrome确实会对localhost上的Web服务器遵循CORS头。如果你是Web开发者，你可能在使用Create React App时遇到过这种情况：前端应用在一个端口上，后端API在另一个端口上。你的应用在对localhost发起跨域请求，这在所有浏览器中都是支持的。

这告诉我，Zoom可能急于推出这个功能，但并不理解CORS。他们无法在浏览器不阻止的情况下发起AJAX请求。于是，他们构建了这个图片hack来绕过CORS。这样做使Zoom暴露了一个严重漏洞，因为不仅Zoom网站可以触发原生客户端的操作并获取响应，互联网上的每个其他网站也可以这样做。

那么，这个功能的安全实现应该是什么样的呢？监听 `localhost:19421` 的Web服务器应该实现一个REST API，并设置 `Access-Control-Allow-Origin` 头，值为 `https://zoom.us`。这将确保只有运行在zoom.us域名上的JavaScript才能与本地Web服务器通信。此外，为了防止页面在后台自动打开Zoom会议，zoom.us应该设置一个内容安全策略（Content Security Policy）头来阻止在iframe中渲染。

这仍然存在一个漏洞：任何页面都可以将你的浏览器重定向到一个你没有预期的Zoom会议链接，但这是Zoom做出的用户体验决策，而非软件漏洞。个人而言，我认为这里的方式也是错误的。他们提到希望通过直接打开应用程序来获得更好的用户体验，但良好用户体验设计的规则之一是：你的软件必须是可预测的。

如果我点击一个链接，我期望它不会突然把我的摄像头和麦克风暴露给我不认识的人。Zoom违反了这个预期。即使他们出于用户体验原因不想使用浏览器内置的弹窗，也应该把这个弹窗放在应用内！Google Meet在这方面做得很好。

我不想偏离本文关于CORS的重点。无论用户体验方面的争论如何，在localhost上运行Web服务器本身就是一项有风险的做法。它绝对不应该向互联网上的每个网站提供特权功能的访问权限，比如安装软件。CORS使你能够安全地做到这一点——不要试图绕过它！

我无法确定是否因为不理解CORS才导致Zoom以这种方式实现了该功能。但是，我与几个人讨论后，我们都无法找到任何正当理由来解释他们现有方式的合理性。在Reddit上，lerunicorn确实发现并指出Firefox可能会阻止从安全来源到非安全来源的XHR请求，这或许能解释这种方式背后的动机。然而，当来源是localhost时，Firefox是支持这一点的。此外，原生应用可以生成唯一的自签名证书。另外，他们也可以使用浏览器扩展。在任何可能的情况下，这都不是忘记过滤来源的正当理由。

不仅仅是Zoom。根据我的观察，很多与我交谈过的开发者都不太理解CORS的工作原理。Stack Overflow上也有大量相关问题的例子。不幸的是，这些问题通常伴随着推荐非常不安全的默认配置的页面，比如express中的这个例子，如果原样复制会使你的应用变得脆弱。其他供应商也被发现存在与Zoom完全相同的漏洞。

开发者只是想让代码能运行，而完全绕过同源策略可能确实能让代码运行，但当有人发现你的做法时，你就会遇到Zoom现在面临的问题。

我见过经验丰富的开发者和新手开发者都对CORS感到困惑。CORS API是否过于复杂和令人困惑，还是我们只需要在CORS和CSP等问题上提供更好的开发者教育？我不确定，但目前的状况显然是行不通的。
