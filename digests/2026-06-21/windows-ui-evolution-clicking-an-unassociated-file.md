让我们打开文件管理器，点击一个既不是 `.EXE`、也不是系统已知类型（如 `.TXT`）的文件。会发生什么？

我手头并没有每一个版本的 Windows，只有其中几个。我们来逐个看看。

另外，我把一些德语标签翻译成了英文，但原始英文版本中的按钮名称可能略有不同。

在 **Windows 386/2.11**（1989）中，你只会得到一句"不行"（图片翻译："ABC.OMG 不是可执行文件"）：

这个版本的 Windows 已经有了"用程序 `$bar` 打开文件类型 `$foo`"的概念，但似乎没有配置此功能的界面。或者说我没找到。你可以在 `WIN.INI` 中设置文件关联：

**Windows 3.1**（1992）情况略有改善：

它仍然只告诉你"不行"，但现在还会提示你存在文件类型关联的概念，并告知在哪里进行配置。由此可以进入一个基本但功能完备的对话框：

顶部的文本框可输入文件扩展名，底部的列表框可选择已安装的程序。如果目标程序不在列表中，可以点击"Durchsuchen..."（"浏览..."）按钮来选择任意程序。

**Windows NT 3.1**（1993）基本相同（Windows for Workgroups 3.11 也是如此）：

**Windows 95**（1995）是一次重大进步，因为现在会弹出一个对话框，让你直接在当前界面选择想要的程序——不再需要跳转到其他地方：

如果程序不在列表中，可以点击"Andere..."（"其他..."）按钮来直接选择程序。

这一行为在 Windows 98、Windows ME 和 Windows 2000 中保持不变：

然后 **Windows XP**（2001）来了，它开始试图引导你使用某种网络服务，因为互联网"终于"无处不在™了：

遗憾的是，我已经记不清那是什么网络服务了。如果你知道，请告诉我。它很可能由微软运营，但它的数据库是什么？列出了哪些程序？它如何查询数据库——是上传整个文件、文件的部分内容还是仅上传文件扩展名？它是否提供在你的电脑上安装程序的功能？（我隐约记得这个服务曾被当作某种攻击载体利用，杂志上写过"不要用这个！！1！"之类的警告，但我不太确定了。）

——编辑：好的，一位读者告诉我，这个功能只是在 Internet Explorer 中打开一个网站，将文件扩展名作为查询字符串发送。据说该服务自 2006 年起已关闭。而且显然相当没用。该读者推测只有微软认证合作伙伴的产品才会出现在这里，并指出即使是像 `.3GP` 这样流行的文件类型也无法被识别。你可以自己去 Internet Archive 上试试看：

现在有一个很大的断层。我没有 Windows XP 和 Windows 10 之间任何版本的系统。所以接下来是 **Windows 10**（2015）：

哎呀，我们显然已经进入了"一切皆扁平化"的时代，你已经很难分辨屏幕上哪些元素是可交互的。连窗口装饰都没了。顶部的文字（"Im Store nach einer App suchen"——"在应用商店中搜索应用"）实际上是一个可点击的按钮（那个图标大概是在暗示这一点？）。"Weitere Apps"（"其他应用"）也是可点击的（通过颜色表示，像网页链接一样），点击后会展开一个列表：

同样，这里显示的是已安装的程序。抱歉，现在叫"应用"了。你可能不会立即注意到的是，这个窗口可以滚动以显示第三个选项：

如果列表中没有你想要的程序，可以点击那个按钮/文字/不管它是什么"Andere App auf diesem PC suchen"（"在此电脑上查找其他应用"）。
