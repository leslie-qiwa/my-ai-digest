# Ask HN: MacBook vs. Dedicated GPU for LLM

对于在 MacBook 上使用大语言模型（LLM）的人来说，想了解 MacBook 与独立 GPU 在运行这些模型时有什么不同吗？以及如何知道一台 MacBook 能运行多大的模型？

我问了几个做机器学习工程师的朋友这个问题，他们都说要用云端及其基础设施来跑这些 LLM，因为那样会快得多。

如果你只是想随便玩玩，我建议看看 @JSR_FDD 的评论。

接下来你大概就要告诉我：汽车销售员都建议直接租新车，医生都建议直接遵循标准疗法，建筑工人都建议把活儿外包出去，而你那些技术圈的朋友都说直接用 AWS 吧。

我卖了十年车，连贵得离谱、却便宜卖给员工的展示样车都没要，一直开着我那辆十年的旧车。

不过话说回来，大多数汽车销售员都有点蠢，虽然最优秀的那些至少算正常。（我是加州大学机械工程专业毕业，全国前 0.2% 的汽车销售员。）他们还会慢慢地自我洗脑，逐渐相信自己说的大部分话。

如果你本来就想要一台大配置的 MacBook，那它很棒。它们跑本地 LLM 还不错，跑本地图像模型很出色，而且既然是 MacBook，有 AppleCare+ 给你撑腰。在我看来，如果你本来就想买 MacBook，那它毫无疑问值得入手；但如果你买它的理由是为了跑 LLM，那就是个糟糕的选择。

我同意。要跑一个可以接受的模型（例如 Qwen/Qwen3.6-27B 或 google/gemma-4-31B），加上不错的量化（最低 Q5）和不错的上下文长度（最低 64k），你可以买 2 块甚至 3 块 GTX 5060 16GiB 显存的显卡，每块约 550 美元。顺带一提，那些快得多的 MoE 模型对我的使用场景毫无用处——比如无法正确区分“我/你”，陷入无尽的思考循环等等。

我目前用 RTX 5070 12GiB + RTX 5060 16GiB + RTX 3060 12GiB 来跑这些模型，上下文长度 96k，配合 MTP/推测解码（speculative decoding），我相当满意（5070 大约比 3060 快 4 倍，5060 介于两者之间，所以大约比 3060 快 2 倍）。

我在中国度假时买了两块 20GB 的 RTX3080（花了我 700 欧元），用 Qwen 3.6 27b Q8（MTP、P2P、200k 上下文）能跑到 800-1000 的输入 tps 和 60-100 tps 的输出。这在编程时（pi harness）感觉达到了 opus4.5 的水平。

而且用这种方式从家里搭建自己的 OpenAI 兼容 API 也很容易，同时还能把 MacBook 当作开发工作站。

MacBook M5 64GB——可以运行 gemma-4-26b-a4b-it-4bit 和 Qwen3.6-35B-A3B-4bit，使用 MLX，在最高 100K token 的上下文下达到约 1500 tps 的前缀处理（prefix）和 45 tps 的解码（decode）。它比 Claude 还快。我真的很惊讶，gemma4 的聊天质量也和 Claude 相近。智能体（Agentic）能用，但比不上云端模型，你仍然可以做那种顶层是代码的智能体。

抱歉再问一次：MacBook 里 GPU 实际能用多少内存？因为内存是共享的（操作系统和应用程序也得用同一块内存）？这跟独立 GPU 的显存不一样吗？

它是完全共享的，所以操作系统和其他一切大概会占用 8GB 左右的 RAM。在一台 64GB 的机器上，你可以运行约 45GB 大小的模型，同时还留有空间让这些模型运行时执行其他本身也可能需要内存的任务。对用户来说，GPU 看起来就是按需占用 RAM，跟系统上运行的任何其他进程一样。你可以在“活动监视器”（或 htop）里看到你的 LLM 占用了多少空间，以及它们用了多少 GPU 容量（全部）。

我不明白你为什么因为问个问题就被踩。视具体规格而定，它们_可能_为更大的模型提供更多余量，但仍然会受限于 CPU 及其相关的总线速度。

那款 OLED 触摸屏 MacBook 据传现在要叫 MacBook Ultra，而且已经被推迟了好几次。它的价格大概会和一辆不错的自行车差不多。

这也是我的看法。即使你买了像 8 块 GB10 的集群或 4 块 A100 这样的硬件，它们仍然会很慢，而且比你习惯用的模型稍微笨一点。我们还得再等等更好的硬件。很多公司都在推进前沿，希望它很快就会到来。

竞争和创新有望让泡沫破裂，到时我们就能拿到价格合理、能运行非常智能模型的本地硬件。像是搭配 GLM 5.2 的 Talaas 那样的东西会很酷。或者苹果把最新模型印进硬件里——那会给每年买一台新 Mac 提供新的理由（每出一个新版本就配一个新的 AI 模型）。

对于愿意容忍轻微延迟的人来说，硬件今天就已经在这里了。人们很容易忘记，计算任务过去往往要花大量时间——渲染一个音频文件、渲染一段视频、转码——各种任务都要花几分钟甚至几个小时，计算机风扇全速狂转，就为了交付一个结果。AI、智能体 AI 以及扩散模型是下一轮这样的东西——用你的一小点等待时间换取惊人的算力。那些想让你对 LLM 平台即时响应上瘾的数据中心建设者，让你误以为“好的”AI 就该即时、完全交互地响应——其实它们带点延迟也依然可以很出色。而且让一个称职的智能体在我的本地机器上做事情，它花十分钟、一小时还是六小时完成一项任务，对我来说其实无所谓，反正我出门干别的事去了。

嗯，我有 A100 和 GB10 的使用权限，但如果我用托管在那上面的模型来编程，我会浪费很多时间等答案和纠正错误。前沿托管模型的质量和速度让我效率高得惊人，还有大量空闲时间。我本可以用那套慢吞吞的本地配置，但代价是什么呢？

那么，如果这一切都被从你手里拿走，你得去银行借钱来重建、好让自己恢复到现在这样的生产力，那要花多少钱，银行又会不会把钱借给你呢？

用 M5 16 核 48GB 加 Qwen 3.6 35B Q4，我能跑到最高 1900 PP/s 和 80 TG/s。
用一块 Nvidia 5090，我能跑到 7800 PP/s 和 280 TG/s。
配合 pi mono，我可不想回到 Claude 之类的东西了。速度、答案质量、一天中任何时候都很短的响应时间——一旦你尝过这果子，你对 SOTA（业界最佳）的定义就会改变……
作为参考，我做软件开发已经 30 年了，我可不是在“氛围编程”（vibe coding）做第无数个待办清单。

MacBook（以及一般的 Mac）对 LLM 来说是个不错的组合，因为它们自带这么多 RAM，
而对 LLM 来说，更多 RAM 意味着能用上更好的模型。
MacBook 也许不如同等 RAM 容量的 GPU 快，但更实惠且整合度高。
最后但同样重要的一点：与 PC+GPU 相比，MacBook 要么是静音的（Air），要么至少在你在意噪音时没那么烦人。
追求极致灵活性和低噪音：在你需要/想要时使用云端 GPU，如果你没有需要 7×24 小时运行的工作负载，这大概也是最具性价比的方案。

使用独立 GPU 时，延迟在于把数据传输到 GPU。在 ARM 上你没有这种延迟。
但这真的取决于你想做什么。一个经过 MLX 优化的较新模型会运行良好、速度不错。比如 Granite4.1（几个月前的模型）只占用 2GB 内存，快得惊人，效果比 gpt-oss-120b（一年前的模型）这种大得多的模型还好。它甚至能在 M1 Mac 上以不错的速度运行。

大约在二月或三月，我开始研究硬件选项，想着手学习训练模型和与模型打交道。我的预算有限，而一台苹果官翻的 32GB Mac mini 在我的预算范围内是遥遥领先的最佳选择。我希望它能更快，但我可以让它 7×24 小时运行，没有噪音，功耗也极低。我只是把长时间运行的任务安排在我睡觉或上班的时候。然后还有一个巨大的加分项：我有了一台超棒的日常主力机，想干别的什么都行。

MacBook 有大量 RAM 且没有 PCIe 瓶颈，但每秒浮点运算次数（FLOP/s）大约比一块便宜得多的 Nvidia GPU 少 10 倍。在 vast.ai 或其他类似服务上用租来的 GPU 测试 LLM（当心存储等问题）。在你尝试并确切知道自己能得到什么之前，别花上千块钱。
