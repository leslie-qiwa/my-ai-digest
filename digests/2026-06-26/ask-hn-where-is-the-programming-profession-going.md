# Ask HN: Where is the programming profession going?

我经营一家小型（3人）软件公司大约有4年时间。自从公司关闭后，我最近去一位朋友的公司（15人）转了转，看看他们在做什么。先说明一下：我是Claude的重度用户（很少手写代码），但我亲眼所见的情况让我相当震惊，我想和其他人校准一下看法。

具体来说：
- 代码不再是唯一可信来源；而是让Claude来写，再让Claude来解释
- 代码行数（LoC）、抽象，以及所有那些"软件开发原则"，对人们来说似乎都不重要了
- 代码审查不再由人来做
- 真正深入理解问题似乎也被交给了Claude
- 一些开发者同时运行5个以上的Claude会话，而且没有人去看代码
- LLM生成的测试爆炸式增长

首先，这和你们公司的情况类似吗？

如果这家公司具有代表性，那么感觉软件开发正在从一种需要高度理解的精确职业，转变为某种概率性的、把理解外包出去的东西（说实话，最终甚至可能不再是一种职业）。

我很想听听其他人的看法。

理解问题，并充分理解现有系统，以便设计出正确的方案——即使有AI辅助——也是更高的认知负担。我最近做这类事情多了很多。

我效率更高了，但也更累了。这可能部分是因为我的团队所负责的范围很广，这让我的一天比其他团队更需要频繁切换上下文。

正如这个帖子里其他人指出的，情况仍在不断演变。然而，我对被AI取代的担忧每天都在减少。在我的经验里，工作量总是超过可用的人力带宽。

对我来说显而易见的是，对速度和产出的期望将会提高（而且正在提高）。要满足这些期望，使用AI将成为必需。学会有效地使用这个新工具，对于职业发展（和职业保全）将至关重要。
